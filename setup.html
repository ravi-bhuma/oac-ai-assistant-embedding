Module 01

Setup Guide

Build a working embedded OAC AI Assistant from a blank file to a running page in six steps. No frameworks, no build tools — just a text editor and Python.

💡
Why start from scratch? In most real-world scenarios, you already have an existing portal, intranet, or web application — you won't be creating a new HTML page from a blank file. This quick-start builds a standalone page so you can learn each piece in isolation, verify it works, and understand what each component does. Once you've completed this module, you'll know exactly which code fragments to place into your existing application:
  • The <script> tag for embedding.js goes in your page's <head> section
  • The CSS rules merge into your existing stylesheet
  • The <oracle-analytics-assistant> tag drops into wherever you want the Assistant to appear in your page layout
  • The applyBindings() init script goes at the bottom of your page or in your app's initialization logic
Step 1

Create the HTML Skeleton

Every HTML page starts with a skeleton — the bare minimum structure a browser expects. You will add the OAC pieces on top of this in subsequent steps.

Open your text editor and create a new file named oac-ai-assistant.html. Paste the following code:

<!DOCTYPE html>
<html dir="ltr">
<head>
  <meta charset="utf-8">
</head>
<body>
</body>
</html>
ℹ️
Why dir="ltr"? This attribute controls text layout direction. OAC's embedded components require it to render correctly. Use ltr (left-to-right) for English and most Western languages. For Arabic or Hebrew, use rtl. Never use auto — it is not supported.
✅
Checkpoint: Open the file in Chrome. Press F12 to open DevTools, click the Elements tab, and confirm you see <html>, <head>, and <body> with no red error messages in the Console tab.
Step 2

Load the Embedding Library

Oracle Analytics Cloud ships a JavaScript library (embedding.js) that draws OAC components inside other web pages. You load it with a single <script> tag. Think of it as "plugging in" OAC to your page.

Add the following inside your <head>, right after the <meta charset> line:

<script
  src="https://<your-oac-host>/public/dv/v2/embedding/auto/embedding.js">
</script>

Replace <your-oac-host> with your actual OAC instance URL. For example:

https://oacp-idseylbmv0mm-ia.analytics.ocp.oraclecloud.com/public/dv/v2/embedding/auto/embedding.js
ℹ️
Embedding modes: The URL path includes an embedding mode. Use auto to let OAC auto-detect JET version differences (recommended for APEX). Use standalone if not using Oracle JET. Use jet only if your host app is already a JET application. The v2 in the path is the embedding framework version.
❌
Common Mistake: Do not download embedding.js locally. Always reference it from your OAC instance — it is updated with each OAC patch and must stay in sync.
✅
Checkpoint: Open DevTools → Network tab, refresh the page, and filter for "embedding.js". You should see it listed with a 200 status. If it shows 404, check the URL for typos.
Step 3

Set Up the Layout

Without a few lines of CSS, the Assistant panel can end up tiny or cause double scrollbars. These styles make it fill the entire browser window.

Add a <style> block right after the </script> tag, still inside <head>:

<style>
  html, body {
    margin: 0;
    height: 100%;
  }
  oracle-analytics-assistant {
    width: 100%;
    height: 100%;
    display: block;
  }
</style>

Why each line matters:

  • margin: 0 — removes default browser margins so the Assistant touches the edges.
  • height: 100% — stretches both <html> and <body> to full viewport height.
  • oracle-analytics-assistant — a custom element that OAC creates. You style it like any HTML tag.
  • display: block — custom elements default to inline; block makes them fill the width.
Step 4

Embed the AI Assistant

Add the <oracle-analytics-assistant> tag inside your <body>. The src-data attribute tells the Assistant which data source to connect to.

⚠️
Your data source must be indexed for search in OAC before it can be used with the embedded AI Assistant. If the data source is not indexed, the Assistant will not be able to query it.

The format of src-data depends on your data source type. In each example below, the green highlighted portion is the Object ID — this is the part you replace with your own value.

RPD Subject Area — the Object ID is the plain display name

<oracle-analytics-assistant
  src-data="[&quot;A - Sample Sales&quot;]">
</oracle-analytics-assistant>

File-based Dataset — the Object ID contains the owner + dataset name (check your Inspect panel)

<oracle-analytics-assistant
  src-data="[&quot;XSA('<owner>'.'<dataset-name>')&quot;]">
</oracle-analytics-assistant>

Local Subject Area (LSA) — the Object ID is a system GUID + LSA name

<oracle-analytics-assistant
  src-data="[&quot;XSA('abda89d7-e772-4efa-b715-6e7c250b0201'.'SampleSalesDS')&quot;]">
</oracle-analytics-assistant>

How to Find Your Object ID

Every data source in OAC has an Object ID. Here’s how to find it:

  1. In OAC, navigate to Data and locate your dataset or subject area.
  2. Right-click (or use the action menu) and select Inspect.
  3. In the Inspect panel, look for the Object ID field — this is the value you need.
  4. Copy it and use it as the green highlighted placeholder in the examples above.

Alternatively, open any workbook that uses the data source and check the browser URL — the subjectareas= parameter contains the full reference.

⚠️
Always wrap values in &quot; — especially for names with spaces or special characters. Do not replace &quot; with regular double quotes in the HTML. The id attribute on the tag is optional — only needed if you need JavaScript access to the element.
Step 5

Initialize (Turn It On)

The Assistant tag is in the page but not active yet. This script "wakes it up" by telling the embedding library to find all OAC components and connect them.

Add this script after the </oracle-analytics-assistant> tag, still inside <body>:

<script>
  oracle.oa.embedding.ready().then(function (oApp) {
    oApp.applyBindings();
  });
</script>

What happens here:

  • oracle.oa.embedding.ready() — waits until the embedding library has fully loaded.
  • .then(function (oApp) { ... }) — runs your code once the library is ready.
  • oApp.applyBindings() — scans the DOM for all <oracle-analytics-assistant> (and <oracle-dv>) tags and activates them.
❌
Ordering matters: This script must come after the <oracle-analytics-assistant> tag in the HTML. If you place it before, applyBindings() runs before the element exists and nothing renders.
Step 6

Serve Locally with Python

Browsers block embedded analytics when you open an HTML file directly via file://. You need to serve it from a local web server. Python includes one out of the box — but first, let's make sure Python is installed.

6a — Check if Python Is Already Installed

Open a terminal (Mac/Linux) or Command Prompt / PowerShell (Windows) and run:

On Windows:

python --version

On macOS / Linux:

python3 --version

If you see something like Python 3.14.2 (any 3.x version), you're good — skip to 6c.

⚠️
Windows uses python, not python3. If you see 'python3' is not recognized as an internal or external command', that's normal on Windows — use python instead. Throughout this guide, replace python3 with python on Windows.
ℹ️
If neither python nor python3 returns a version number, Python is not installed. Follow Step 6b below.

6b — Install Python (If Not Present)

Windows:

  1. Go to python.org/downloads and click Download Python 3.x.
  2. Run the installer. Important: Check the box that says "Add python.exe to PATH" at the bottom of the first screen — this is the most common mistake.
  3. Click Install Now and wait for it to finish.
  4. Close and reopen your Command Prompt, then verify: 
    python --version
# Should show: Python 3.x.x

macOS:

  1. macOS often ships with Python 3 pre-installed. If not, the easiest way is via Homebrew: 
    # Install Homebrew (if not already installed)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install Python
brew install python
  2. Verify: 
    python3 --version

Linux (Ubuntu/Debian):

sudo apt update
sudo apt install python3 -y
python3 --version

6c — Test Python Works

Before serving files, do a quick sanity check to confirm Python runs correctly:

On Windows:

python -c "print('Python is ready!')"

On macOS / Linux:

python3 -c "print('Python is ready!')"

You should see Python is ready! printed. If so, you're all set.

6d — Start the Local Web Server

Navigate to the folder where your HTML file is saved, then start the server:

On Windows:

cd C:\Users\YourName\Desktop\oac-workshop

python -m http.server 8000

On macOS / Linux:

cd ~/Desktop/oac-workshop

python3 -m http.server 8000

You should see output like:

Serving HTTP on :: port 8000 (http://[::]:8000/) ...

As you open pages in Chrome, the terminal will log every request:

Serving HTTP on :: port 8000 (http://[::]:8000/) ...
::1 - - [11/Mar/2026 18:51:44] "GET /oac-ai-assistant.html HTTP/1.1" 200 -
::1 - - [11/Mar/2026 18:53:56] "GET /oac-ai-assistant.html HTTP/1.1" 200 -

Reading the log: A 200 means the file was served successfully. If you see 404, it means the file was not found — check your filename and make sure you're in the right folder.

❌
The server must stay running. This is the most common beginner mistake — the server only works while the terminal window is open and the process is active. If you close the terminal, minimize it and forget, or accidentally press Ctrl + C, the server stops and your page will show a "connection refused" error in Chrome. If that happens, just re-run the same python -m http.server 8000 command to start it again.

6e — Open the Page in Chrome

Open Google Chrome and navigate to:

http://localhost:8000/oac-ai-assistant.html

You should see a blank or loading page — that's expected until you add the Safe Domain in OAC (next sub-step).

6f — Register the Safe Domain in OAC

OAC will block the embedded Assistant unless your local server origin is registered:

  1. In a new Chrome tab, sign in to your OAC instance.
  2. Click the hamburger menu and go to Console → Administration → Safe Domains.
  3. Click Add Domain and enter exactly: http://localhost:8000
  4. Check the Embedding checkbox.
  5. Click Save.
❌
The origin must match exactly — protocol (http, not https), hostname (localhost), and port (8000). If you use a different port, update the Safe Domain entry accordingly.

6g — Verify the AI Assistant Loads

  1. Go back to your http://localhost:8000/oac-ai-assistant.html tab.
  2. Refresh the page.
  3. If using browser-based auth, make sure you are logged into OAC in another tab.
  4. The AI Assistant chat interface should appear and begin loading.
✅
Checkpoint: The URL bar shows http://localhost:8000/oac-ai-assistant.html (no file://). The AI Assistant renders and you can type a question. If the Assistant doesn't appear, open DevTools (F12) → Console tab and check for CORS or 404 errors.

Alternative: VS Code Live Server

If you use VS Code, you can skip the Python server entirely. Install the Live Server extension, right-click your HTML file, and select "Open with Live Server". Default port is 5500 — add http://localhost:5500 to OAC Safe Domains instead.

Next

Embed into Your Existing Application

Now that you've seen how the standalone page works, here's how to integrate the AI Assistant into your existing portal or web application. You need to place four code fragments into the right locations:

1. Add the embedding library to your <head>

Place this alongside your existing <script> and <link> tags. It does not conflict with other libraries.

<script
  src="https://<your-oac-host>/public/dv/v2/embedding/auto/embedding.js">
</script>

2. Add CSS for the assistant container

Merge these into your existing stylesheet. Adjust width and height to fit your layout — it doesn't have to be 100%. For example, a sidebar panel might use width: 400px; height: 100vh;.

oracle-analytics-assistant {
  width: 100%;
  height: 600px;  /* or any size that fits your layout */
  display: block;
}

3. Place the assistant tag in your page

Drop it into any <div> container in your existing page layout — a sidebar, a tab panel, a modal, or a dedicated section. Use the appropriate src-data format for your data source type (see Step 4 above).

<!-- RPD Subject Area -->
<div class="your-assistant-container">
  <oracle-analytics-assistant
    src-data="[&quot;A - Sample Sales&quot;]">
  </oracle-analytics-assistant>
</div>

<!-- Or for a file-based Dataset -->
<div class="your-assistant-container">
  <oracle-analytics-assistant
    src-data="[&quot;XSA('<owner>'.'<dataset-name>')&quot;]">
  </oracle-analytics-assistant>
</div>

4. Initialize — either inline or in your app's startup

If your application already has an initialization routine (e.g., DOMContentLoaded, Angular ngOnInit, React useEffect), call applyBindings() from there. Otherwise, a <script> tag at the bottom of the page works fine.

<!-- Option A: Inline script at the bottom of your page -->
<script>
  oracle.oa.embedding.ready().then(oApp => oApp.applyBindings());
</script>

<!-- Option B: Inside your app's existing init logic -->
<script>
  document.addEventListener('DOMContentLoaded', () => {
    oracle.oa.embedding.ready().then(oApp => oApp.applyBindings());
  });
</script>
⚠️
Don't forget: Your application's production domain must be registered in OAC Safe Domains with the Embedding checkbox enabled. See Module 04: Reference & Resources (reference.html) for the exact steps.
ℹ️
Authentication in your app: If your portal already uses SSO with IDCS, browser-based auth will work seamlessly — no extra code needed. If your users don't have OAC accounts, use token-based auth instead. See Module 02: Security Deep-Dive (security.html) for both approaches.
