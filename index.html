How Embedding Works

The embedding framework relies on four components. Remove any one and the AI Assistant will not render.

Initialization
Load the Engine: The <script> tag pulls the OAC embedding library from your instance. Do not download it locally.
Layout: CSS removes default margins and stretches the assistant to fill the viewport.
Declare: The <oracle-analytics-assistant> element tells OAC where to render. src-data specifies the data source — a dataset or subject area.
Activate: applyBindings() scans the DOM and wires up all OAC elements. Must run after the tags exist.

Login: User authenticates to OAC — directly or via SSO.
Cookie: IDCS sets a session cookie shared across all browser tabs.
Embedded Page: The framework includes the session cookie with API calls automatically.
Validation: OAC verifies the session and renders the Assistant with the user's security rules.
Token-Based (OAuth 2.0) Authentication
1
🌐
User Visits Embedded Page
No OAC login required
2
⚙
Server Calls IDCS Token API
Client ID + Secret + Service Account
3
🔑
Bearer Token Returned
Injected via setEmbeddingConfig()
4
✅
OAC Validates Token
No cookies needed
No Login: User simply opens the page. No OAC account needed.
Server-Side: Your backend calls IDCS /oauth2/v1/token. Credentials never reach the browser.
Injection: Token passed to setEmbeddingConfig() with eSecurityType: "token".
Cookieless: OAC validates the bearer token. No session cookies — no third-party cookie issues.
Authentication Models

Session-Based vs. Token-Based

Understanding when to use each model is critical for production deployments.

AspectBrowser / SessionToken-Based (OAuth 2.0)
How it worksUser authenticates; session cookie shared.Bearer token from IDCS injected via JS.
User login?Yes — active OAC session.No — token obtained server-side.
3rd-party cookies?May break in blocking browsers.None — no cookies used.
Best forInternal portals with SSO.Public sites, kiosks, external portals.
ComplexityLow — Safe Domains only.Medium-High — IDCS app + token endpoint.
04
Data Sources

What Can You Embed?

The embedded AI Assistant works with three types of OAC data sources: RPD Subject Areas, file-based Datasets, and Local Subject Areas (LSA). See Setup, Step 4 for the exact src-data format for each.

For local testing, you'll need Python 3.x and a text editor. See Setup, Step 6 (setup.html) for instructions.
