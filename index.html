Why Embed?

Advantages of Embedding the AI Assistant

The Oracle Analytics Cloud AI Assistant enables users to ask natural-language questions and generate visualized insights on the fly. Embedding it into your own HTML pages unlocks powerful use cases that go far beyond the default OAC console experience.

🌐

Unified User Experience

Bring conversational analytics directly into your portal, intranet, or custom app — users never leave your branded interface.

👥

Non-Technical Access

Business users explore data through natural language without learning OAC navigation. Adoption goes up, training costs go down.

📈

Dashboard + AI Side-by-Side

Embed the Assistant alongside dashboards so users can ask follow-up questions about what they see in charts and tables.

🔒

Governed & Secure

OAC's role-based security, row-level security, and semantic-layer governance apply to every embedded query.

⚡

Zero Frameworks Needed

A single HTML file, one script tag, and one custom element. No React, no build tools, no npm — just a browser.

🔑

Flexible Authentication

Choose browser-based SSO for internal apps, or token-based auth for public-facing sites — both are fully supported.

Real-World Patterns

How Organizations Use the Embedded AI Assistant

The embedded AI Assistant is not just a standalone chat window — it becomes part of your application's user experience. Here are the patterns demonstrated in the official demo video:

💬

Standalone AI Assistant

Embed the Assistant as a full-page experience — users interact with a clean conversational interface connected to a specific dataset. Ideal for focused data exploration without dashboard clutter.

📊

Assistant Next to a Dashboard

Place the AI Assistant alongside an embedded OAC canvas. Users view charts and tables on one side and ask follow-up questions on the other — no context-switching between screens.

⚙

Dynamic Dataset Selector

Build a custom app that lets users search and select different datasets, then dynamically bind the Assistant to the chosen data. This turns the Assistant into a flexible data exploration tool across multiple sources.

📦

Embedded in Line-of-Business Apps

Integrate the Assistant directly into operational applications — inventory trackers, order management portals, supply chain dashboards — so users ask analytical questions without leaving their workflow.

The embedding framework relies on four components working together. Remove any one of them and the AI Assistant will not render.

Step 1 — Load the Engine: The <script> tag in <head> pulls the OAC embedding library from your instance. This file contains everything needed to render OAC components — do not download it locally.
Step 2 — Layout: CSS removes default margins and stretches the assistant to fill the viewport. Without this, the component may collapse to zero height or create double scrollbars.
Step 3 — Declare the Component: The custom element <oracle-analytics-assistant> tells OAC where to render. The src-data attribute specifies which dataset the AI Assistant connects to.
Step 4 — Activate: applyBindings() scans the DOM, finds all OAC elements, and wires them up. This must run after the component tags exist in the page.

Login: The user authenticates to OAC through the browser — either directly or via SSO (Azure AD, Okta, ADFS + IDCS). This establishes an authenticated session.
Cookie: IDCS sets a session cookie in the browser. This cookie is shared across all tabs in the same browser profile and domain context.
Embedded Page: When the user visits your embedded page, the embedding framework automatically includes the session cookie with its API calls back to OAC. No extra code needed.
Validation: OAC verifies the session and renders the AI Assistant under the user's identity — applying their specific role-based and row-level security rules.

No Login: The user simply opens the page. Unlike browser-based auth, they don't need an OAC account or an active OAC session.
Server-Side Token Request: Your backend calls the IDCS /oauth2/v1/token endpoint using the confidential app's Client ID/Secret and a service account. Credentials never reach the browser.
Token Injection: The access token is passed to setEmbeddingConfig() with eSecurityType: "token". The embedding framework uses this token for all subsequent API calls to OAC.
Cookieless Auth: OAC validates the bearer token and renders the Assistant. No session cookies are created — eliminating third-party cookie issues entirely. Token refresh happens server-side.
Authentication Models

Session-Based vs. Token-Based

Oracle Analytics supports two authentication models for embedded content. Understanding when to use each is critical for production deployments.

Aspect Browser / Session-Based Token-Based (OAuth 2.0)
How it works User authenticates to OAC via browser; session cookie is shared with the embedded page. A bearer token (from IDCS/IAM) is injected via JavaScript; no browser login required.
User login required? Yes — user must be logged into OAC in another tab (or SSO handles it). No — the token is obtained server-side and passed to the page.
Third-party cookie issue? Possible — browsers that block 3rd-party cookies may break the session. None — no session cookies are created on the client.
Best for Internal portals where users already have OAC accounts and SSO. Public-facing sites, kiosks, portals with non-OAC users.
Security model OAC session + IDCS SSO policies. OAuth 2.0 bearer token + IDCS confidential app + scoped permissions.
Complexity Low — no extra server-side code. Medium-to-High — requires IDCS app setup, token generation, refresh logic.
Prerequisites

Before You Begin

Confirm you have the following items ready before starting the workshop modules.

🌎

OAC Instance

Oracle Analytics Cloud instance with the AI Assistant and embedding framework available (March 2026 or later recommended).

💻

Browser

Google Chrome recommended. DevTools (F12) will be used for debugging.

🐍

Python 3.x

Any Python 3 version — used only to run a one-line local web server. No Python knowledge required.

✎

Text Editor

VS Code, Notepad++, Sublime, or even plain Notepad. Anything that can save a .html file.

👤

OAC Permissions

User must have the Use Assistant in Workbooks permission (included in DV Content Author role by default).

🗃

Data Source

A dataset or subject area available in your OAC instance for the Assistant to query.
