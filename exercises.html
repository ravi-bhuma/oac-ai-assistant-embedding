Hands-On Exercises

Three guided labs — from standalone AI Assistant embedding to side-by-side dashboard + assistant and token-based authentication testing.

Lab 1 — Beginner

Embed the Standalone AI Assistant

Build a single HTML file that loads the OAC AI Assistant full-screen, using browser-based authentication.

Your Configuration Values

These values will be used throughout the labs. Replace them with your own environment details:

OAC Instance URL:   https://<your-oac-host>.analytics.ocp.oraclecloud.com
OAC User:           your.name@company.com
Dataset Name:       Your_Dataset_Name
Local Test Port:    8000

Instructions

  1. Create a new folder on your desktop called oac-workshop.
  2. Using your text editor, create a file named oac-ai-assistant.html and build it following the six steps from Module 01: Quick-Start Setup (setup.html).
  3. Open a terminal, navigate to the folder, and start the Python server: 
    # Windows:
cd C:\Users\YourName\Desktop\oac-workshop
python -m http.server 8000

# macOS / Linux:
cd ~/Desktop/oac-workshop
python3 -m http.server 8000
  4. In OAC, add http://localhost:8000 to Safe Domains (Administration → Safe Domains → Add Domain → check Embedding).
  5. Open Chrome and navigate to http://localhost:8000/oac-ai-assistant.html.
  6. In a separate tab, log into your OAC instance (this establishes the session cookie).
  7. Return to your embedded page and refresh. The AI Assistant should appear.

Test Questions

Try asking the Assistant these questions to verify it's connected to your data:

  • "Show me a summary of the data"
  • "What are the top 5 values by revenue?"
  • "Show trends over time"
  • "Compare categories"
✅
Checkpoint: The AI Assistant responds to at least two questions with data-backed answers. If it says "no data found," verify the dataset name in src-data matches exactly and that you are logged into OAC.
Lab 2 — Intermediate

Embed Dashboard + AI Assistant Side-by-Side

Create a split-screen layout with an OAC dashboard on the left and the AI Assistant on the right.

Instructions

  1. Create a new file called dashboard-assistant.html in the same folder.
  2. Use the following layout code as your starting point:
<!DOCTYPE html>
<html dir="ltr">
<head>
  <meta charset="utf-8">
  <script
    src="https://<your-oac-host>/public/dv/v2/embedding/auto/embedding.js">
  </script>
  <style>
    html, body { margin: 0; height: 100%; }
    .container {
      display: flex;
      height: 100%;
    }
    .analytics {
      flex: 2;
      height: 100%;
      border-right: 1px solid #ddd;
    }
    .assistant {
      flex: 1;
      height: 100%;
      min-width: 360px;
    }
    oracle-dv, oracle-analytics-assistant {
      width: 100%;
      height: 100%;
      display: block;
    }
  </style>
</head>
<body>
  <div class="container">
    <div class="analytics">
      <oracle-dv
        project-path="<your-project-path>"
        active-page="insight"
        active-tab-id="snapshot!canvas!1">
      </oracle-dv>
    </div>
    <div class="assistant">
      <oracle-analytics-assistant
        id="oaAssistant"
        src-data="[&quot;XSA('user'.'dataset')&quot;]">
      </oracle-analytics-assistant>
    </div>
  </div>
  <script>
    oracle.oa.embedding.ready().then(oApp => oApp.applyBindings());
  </script>
</body>
</html>
  1. Replace <your-project-path> with the path to an OAC workbook (e.g., /@Catalog/shared/MyDashboard). You can find this in OAC under the workbook's Developer → Embed tab.
  2. Replace the src-data values with your user and dataset name.
  3. Serve and test the same way as Lab 1.
ℹ️
Why side-by-side? This pattern is powerful for enterprise analytics — users can see a dashboard and simultaneously ask the AI Assistant follow-up questions about the data they're viewing. It reduces context-switching and enables deeper exploration.
Lab 3 — Advanced

Token-Based Authentication

Modify your Lab 1 page to use a hardcoded token (for testing), simulating the production token-based flow.

Instructions

  1. First, obtain a test token using curl (from Module 02: Security Deep-Dive (security.html), Step B-4): 
    echo -n "<Client-ID>:<Client-Secret>" | base64

curl --request POST \
  --url https://<IDCS-domain>.identity.oraclecloud.com/oauth2/v1/token \
  --header 'Authorization: Basic <base64-string>' \
  --header 'Content-Type: application/x-www-form-urlencoded;charset=UTF-8' \
  --data 'grant_type=password&username=<user>&password=<pass>&scope=<scope>'
  2. Copy the access_token value from the JSON response.
  3. Create a new file called oac-token-auth.html and copy your Lab 1 HTML into it.
  4. Replace the initialization script with the token-based version:
<script>
  const token = 'paste-your-access-token-here';

  oracle.oa.embedding.ready().then((oApp) => {
    oApp.setEmbeddingConfig({
      oSecurityConfig: {
        eSecurityType: "token",
        oOptions: {
          credentials: "",
          tokenAuthFunction: new Function("return '" + token + "'")
        }
      }
    });
    oApp.applyBindings();
  });
</script>
  1. Serve and open the page. This time, you should not need to be logged into OAC in a separate tab — the token handles authentication.
  2. Test the Assistant with the same questions from Lab 1.
⚠️
Token expiry: The default OAC access token expires after 100 seconds. If the Assistant stops working, your token has likely expired — generate a fresh one with the curl command. In production, your server-side endpoint handles refresh automatically.
✅
Checkpoint: The AI Assistant renders and responds to questions without you being logged into OAC in any browser tab. Open an incognito window to verify — the token alone should be sufficient for authentication.

Setting Up a Local Python Environment

Here is a detailed guide to setting up a proper local development environment for testing your embedded HTML pages.

Option A: Minimal Setup (No Virtual Environment)

If you just need a quick server to test:

On Windows:

# Check Python version
python --version

# Navigate to your project folder
cd C:\Users\YourName\Desktop\oac-workshop

# Start the HTTP server
python -m http.server 8000

# Server now running at http://localhost:8000
# Press Ctrl+C to stop

On macOS / Linux:

python3 --version
cd ~/Desktop/oac-workshop
python3 -m http.server 8000

Option B: Virtual Environment with Live Reload (Recommended)

A virtual environment keeps dependencies isolated and adds live-reload for faster iteration:

On Windows:

# Navigate to your project folder
cd C:\Users\YourName\Desktop\oac-workshop

# Create a virtual environment
python -m venv .venv

# Activate it (Command Prompt):
.venv\Scripts\activate.bat
# Or (PowerShell):
.venv\Scripts\Activate.ps1

# Your prompt now shows (.venv) — you're in the virtual env

# Install livereload for auto-refresh on file changes
pip install livereload

# Start the live-reload server
livereload --port 8000 .

# To deactivate when done:
deactivate

On macOS / Linux:

cd ~/Desktop/oac-workshop
python3 -m venv .venv
source .venv/bin/activate
pip install livereload
livereload --port 8000 .
deactivate

Option C: HTTPS Local Server (For Testing Token Auth)

Some production token flows require HTTPS. You can use a self-signed certificate (macOS/Linux, or Windows with Git Bash):

# Generate a self-signed certificate
openssl req -x509 -newkey rsa:2048 -keyout key.pem -out cert.pem \
  -days 365 -nodes -subj "/CN=localhost"

# Start an HTTPS server with Python
# Windows: use "python" instead of "python3"
python3 -c "
import ssl, http.server
ctx = ssl.SSLContext(ssl.PROTOCOL_TLS_SERVER)
ctx.load_cert_chain('cert.pem', 'key.pem')
srv = http.server.HTTPServer(('localhost', 8443), http.server.SimpleHTTPRequestHandler)
srv.socket = ctx.wrap_socket(srv.socket, server_side=True)
print('Serving HTTPS on https://localhost:8443')
srv.serve_forever()
"

# Add https://localhost:8443 to OAC Safe Domains
# Browser will show a certificate warning — click "Advanced" then "Proceed"

Project Folder Structure

oac-workshop/
  .venv/                    # Virtual environment (don't commit to git)
  oac-ai-assistant.html     # Lab 1: Standalone AI Assistant
  dashboard-assistant.html  # Lab 2: Dashboard + Assistant
  oac-token-auth.html       # Lab 3: Token-based auth
  cert.pem                  # (Optional) Self-signed cert for HTTPS
  key.pem                   # (Optional) Private key for HTTPS
ℹ️
VS Code users: Install the "Live Server" extension for one-click serving. Default port is 5500. Remember to add http://localhost:5500 to OAC Safe Domains instead of port 8000.
