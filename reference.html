src-data Format Reference

The src-data attribute tells the AI Assistant which data source to query. The green highlighted portion is the Object ID — get it from the dataset's Inspect panel in OAC.

<!-- RPD Subject Area -->
<oracle-analytics-assistant
  src-data="[&quot;A - Sample Sales&quot;]">
</oracle-analytics-assistant>

<!-- File-based Dataset -->
<oracle-analytics-assistant
  src-data="[&quot;XSA('<owner>'.'<dataset-name>')&quot;]">
</oracle-analytics-assistant>

<!-- Local Subject Area (LSA) -->
<oracle-analytics-assistant
  src-data="[&quot;XSA('<GUID>'.'<lsa-name>')&quot;]">
</oracle-analytics-assistant>
💡
How to find the Object ID: In OAC, go to Data, locate your dataset or subject area, right-click and select Inspect. Copy the Object ID from the Inspect panel and use it as the green highlighted placeholder in the examples above. The id attribute on the tag is optional.

Complete Code: Browser-Based Auth

The minimal standalone AI Assistant embedding with session-based authentication:

<!DOCTYPE html>
<html dir="ltr">
<head>
  <meta charset="utf-8">
  <script
    src="https://<your-oac-host>/public/dv/v2/embedding/auto/embedding.js">
  </script>
  <style>
    html, body { margin: 0; height: 100%; }
    oracle-analytics-assistant {
      width: 100%; height: 100%; display: block;
    }
  </style>
</head>
<body>
  <oracle-analytics-assistant
    id="oaAssistant"
    src-data="[&quot;XSA('<user>'.'<dataset>')&quot;]">
  </oracle-analytics-assistant>
  <script>
    oracle.oa.embedding.ready().then(oApp => oApp.applyBindings());
  </script>
</body>
</html>

Complete Code: Token-Based Auth

AI Assistant embedding with OAuth 2.0 token authentication:

<!DOCTYPE html>
<html dir="ltr">
<head>
  <meta charset="utf-8">
  <script
    src="https://<your-oac-host>/public/dv/v2/embedding/auto/embedding.js">
  </script>
  <style>
    html, body { margin: 0; height: 100%; }
    oracle-analytics-assistant {
      width: 100%; height: 100%; display: block;
    }
  </style>
</head>
<body>
  <oracle-analytics-assistant
    id="oaAssistant"
    src-data="[&quot;XSA('<user>'.'<dataset>')&quot;]">
  </oracle-analytics-assistant>
  <script>
    const token = '<access-token>';

    oracle.oa.embedding.ready().then((oApp) => {
      oApp.setEmbeddingConfig({
        oSecurityConfig: {
          eSecurityType: "token",
          oOptions: {
            credentials: "",
            tokenAuthFunction: new Function("return '" + token + "'")
          }
        }
      });
      oApp.applyBindings();
    });
  </script>
</body>
</html>

Configuring Token Expiry

OAC access tokens default to 100 seconds — far too short for interactive sessions. To extend the expiry, use the IDCS SCIM API to patch the OAC application settings:

# Step 1: Get a bearer token for the IDCS admin API
curl --request POST \
  https://<IDCS-domain>.identity.oraclecloud.com/oauth2/v1/token \
  -H 'Authorization: Basic <base64 clientID:clientSecret>' \
  -H 'Content-Type: application/x-www-form-urlencoded;charset=UTF-8' \
  -d 'grant_type=client_credentials&scope=urn:opc:idm:__myscopes__'

# Step 2: Patch the OAC app to extend token lifetime
curl --request PATCH \
  --url https://<IDCS-domain>.identity.oraclecloud.com/admin/v1/Apps/<OAC-AppID> \
  --header 'Authorization: Bearer <admin-token-from-step1>' \
  --header 'Content-Type: application/json' \
  --data '{
    "schemas": ["urn:ietf:params:scim:api:messages:2.0:PatchOp"],
    "Operations": [
      { "op": "replace", "path": "accessTokenExpiry", "value": 3600 },
      { "op": "replace", "path": "refreshTokenExpiry", "value": 86400 },
      { "op": "replace", "path": "allowOffline", "value": true }
    ]
  }'
ℹ️
Finding the AppID: The <OAC-AppID> can be found in the IDCS console under the OAC application's App Details section. This is the IDCS application that represents the OAC instance itself, not the confidential application you created for token generation.

Troubleshooting Guide

Problem Likely Cause Fix
Page is blank / Assistant does not appear Safe Domain not added, or page served from wrong origin Double-check the Safe Domain in OAC Admin. Ensure it matches protocol + host + port exactly (e.g., http://localhost:8000).
CORS error in Console OAC does not recognize the embedding origin Add the origin to Safe Domains with the Embedding checkbox enabled. Sign out and sign back in to OAC.
embedding.js shows 404 in Network tab Typo in the script URL Copy the URL from OAC's Developer → Embed tab. Verify the instance hostname and version path (v2).
TypeError: Cannot read properties of undefined Init script ran before embedding library loaded Ensure the applyBindings() script comes after both the embedding.js <script> and the <oracle-analytics-assistant> tag.
Assistant loads but says "no data" Not logged into OAC, or wrong dataset name Log into OAC in another tab. Verify the dataset name in src-data matches exactly (case-sensitive).
Double scrollbar on the page Missing CSS from Step 3 Add html, body { margin: 0; height: 100%; } and set the assistant element to display: block.
Token auth — "AJAX request failed during bootstrap" Token expired or invalid Generate a fresh token. Check token expiry settings. Verify the scope includes the correct OAC instance.
Token auth — CORS error despite Safe Domain Misleading error — actual issue is the token Check the Network tab → first failed request → Headers. The real error is usually in the response body, not CORS.
Works in Chrome but not Safari Safari blocks third-party cookies by default Switch to token-based auth (no cookies needed) or configure a vanity URL so both domains match.
file:// protocol errors Opening HTML file directly instead of via web server Always serve via python -m http.server or VS Code Live Server. Never use file://.

Required Permissions

The embedded AI Assistant is controlled by OAC's role-based access system. The key permission is Use Assistant in Workbooks, which is included by default in the DV Content Author application role.

If your organization uses custom roles, administrators can create a role with only this specific permission — useful for granting AI Assistant access without full content authoring capabilities.

Verify user roles in the OAC Console under Users and Roles → Application Roles.

Production Checklist: Safe Domains

When you embed the AI Assistant into your existing portal or web application, the production domain must be registered as a Safe Domain in OAC — otherwise the embedded components will be blocked by CORS and the Assistant will not render.

Steps

  1. Sign in to Oracle Analytics Cloud as an administrator.
  2. Navigate to Console → Administration → Safe Domains.
  3. Click Add Domain.
  4. Enter the exact origin of your production site — for example https://portal.yourcompany.com. The protocol, hostname, and port must all match.
  5. Check the Embedding checkbox (required for <oracle-analytics-assistant> and <oracle-dv> tags to render).
  6. Optionally check Scripts if your page also loads additional OAC resources.
  7. Click Save.
⚠️
After adding a Safe Domain, users must sign out and sign back in to OAC for the change to take effect. If you are testing in development alongside production, you will need separate entries for each origin (e.g., http://localhost:8000 for dev and https://portal.yourcompany.com for production).
ℹ️
Wildcard support: OAC Safe Domains support wildcards (e.g., https://*.yourcompany.com) to avoid creating multiple entries for subdomains. Use wildcards carefully — only add the domains you actually need.
